# Lineup — футбол в Telegram

Telegram Mini App для любительского футбола: запись на игры с выбором позиции на поле, составы (автобаланс / вручную / драфт капитанами), живое табло, статистика и лидерборды, QR-оплата взносов со сверкой по игре.

**Стек:** Hono + Hono RPC · Drizzle ORM + libsql (SQLite) · grammY · React + Vite + TypeScript · pnpm workspaces.

```
apps/server     API + бот (Hono, grammY, Drizzle)
apps/web        Mini App (React, Telegram WebApp SDK)
packages/shared zod-схемы и доменные константы
```

## Роли

| Роль | Что может |
|---|---|
| Игрок | записываться (позиция на схеме поля), платить по QR, статистика, MVP, лидерборды |
| Организатор | создавать игры/митапы, ростер и заявки, деление на команды, чек-ин, табло, сверка оплат, возвраты, рассылки, модерация, сезоны |
| Владелец | + роли, настройки платформы (валюта, очки, политики), реквизиты и QR для оплаты |

Деньги через приложение **не проходят**: игроки платят по QR сообщества или наличными, организатор отмечает оплату в сверке.

## Запуск в разработке

```bash
pnpm install
cp .env.example .env            # DEV_AUTH=1 для входа без Telegram
pnpm db:seed                    # демо-данные (17 игроков, игры, история)
pnpm dev                        # сервер :3000 + web :5173
```

Открой http://localhost:5173 — без Telegram появится дев-вход с выбором пользователя (владелец, организатор, игроки).

Тесты и проверки: `pnpm test`, `pnpm typecheck`, `pnpm build`.

## Продакшен (docker compose)

1. Создай бота у [@BotFather](https://t.me/BotFather) → получи `BOT_TOKEN`.
2. Заполни `.env` рядом с `docker-compose.yml`:

```env
BOT_TOKEN=123456:ABC...
JWT_SECRET=<длинная случайная строка>
WEBAPP_URL=https://lineup.example.com
OWNER_TG_ID=<твой Telegram id>   # этот аккаунт станет владельцем при первом входе
```

3. `docker compose up -d --build`

Контейнер слушает `:3000` (статика мини-аппа + `/api` + бот через long polling; вебхуки и второй процесс не нужны). БД и загруженные фото живут в volume `lineup-data`.

4. Направь домен с HTTPS на контейнер (Telegram требует https), например Caddy:

```
lineup.example.com {
    reverse_proxy localhost:3000
}
```

5. В BotFather: **Bot Settings → Menu Button** → укажи `WEBAPP_URL`. Бот делает это и сам при старте, плюс отвечает на `/start` кнопкой открытия.

Готово: `/start` у бота → «Открыть Lineup». Первый вход аккаунта `OWNER_TG_ID` создаёт владельца; роли организаторов раздаются в «Инструменты → Роли и доступы».

## Что делает бот

- `/start` — приветствие + кнопка открытия мини-аппа, menu button
- уведомления: публикация составов, продвижение из листа ожидания, одобрение заявки, напоминания об оплате, перенос/отмена игры, итог матча, рассылки организатора
- напоминание ростеру за 2 часа до игры
- автосоздание игр по регулярному расписанию (Инструменты → Регулярные игры)

## Переменные окружения

См. `.env.example`. Ключевые: `BOT_TOKEN`, `JWT_SECRET`, `WEBAPP_URL`, `OWNER_TG_ID`, `DATABASE_URL` (`file:` или `libsql://` Turso), `UPLOAD_DIR`, `DEV_AUTH` (только для разработки).
